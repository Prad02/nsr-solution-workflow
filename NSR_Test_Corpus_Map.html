H · Different Doc Type · Different Workflow
Vendor Statements
Statements, not invoices. Each one lists the vendor's open invoices for reconciliation. These should never trigger posting; they only feed the Statement Reconciliation workflow.
▼
What it tests: §6.3 Statement Reconciliation Workflow — statement classification at ingestion, match key (Vendor + Invoice # + Invoice Date), discrepancy detection (status mismatch, amount discrepancy, missing invoice), row-by-row UI. Statements must NOT post.
25 vendor statements covering: American Recycling · Butler · Contractor's Disposal · Denali · EBI · Evergreen Recyclers · FLEXPAK · Interstate Waste CT · IWP · Lighting Resources · Martin Custom · Organic Matters · Plummers Disposal · Price Farms Organics · Quincy · Shore Waste · Smurfit Westrock · St Louis Composting · Sun Services · The Compost Company · TOR Cust50614 (Northstar Recycling) · Waste Reduction Recycling & Transfer (Bimbo Bakeries) · Waste Solutions Canada · WR Environmental · Zollett

03 · SOW Mapping

Each test corpus → which SOW feature it validates.

Use this when planning UAT. For every milestone in the SOW, here's which folder of test data exercises it.

Corpus SOW Section Why
A · Production Volume Bulk UAT corpus (cross-cuts M2, M3, M4, M5) Realistic volume + vendor diversity. Used for accuracy benchmarking.
B · Tricky M3 · M4 · M4.3 (multiple) Currency, surcharges, address aliases, Contra — one invoice per tricky case.
C · Total / Balances M3 §2.2.1 · M3 §2.2.9 Invoice Total disambiguation when multiple totals are printed.
D · Supplier Addresses M1 (address-aware indexing) · M3 §2.2.2 Address parsing in unusual layouts; site-location alias resolution.
E · Bill-To Mismatch M5 §5.1 check #10 Validation warning (not hard stop) when Bill-To doesn't match NSR.
F · Flip-to-AR / Contra M4 §4.3 · L3 grouped matching The hardest feature in the SOW. Georgetown set is the gold standard.
G · Email-paired M2 §1.1 (entire pipeline) Multi-attachment, forwarded, body-link, supporting documents.
H · Statements M6 §6.3 (Statement Reconciliation) Different doc type. Must NOT post. Tests classifier accuracy.
J · Reference / Schema M1 (data sync) · M6 (posting) · M4 (matching data) Interface contracts and source-system reference. Not test data.
