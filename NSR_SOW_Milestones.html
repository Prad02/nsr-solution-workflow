Milestone 01

Foundation, SSO, and CieTrade Master Data Sync

Confirmed in commercial proposal
Objective
Stand up the application foundation, establish all CieTrade master data feeds, and enable secure user access. This milestone unblocks every downstream module.
Deliverables
  • Application foundation. Multi-tenant infrastructure, three environments (dev, UAT, production), CI/CD pipeline, monitoring and alerting baseline.
  • Single Sign-On. SAML 2.0 integration with Microsoft 365 / Azure AD. Just-in-time user provisioning. Role claim mapping from Azure AD groups.
  • Counterparty master sync. Daily, with active-only filter and address-aware indexing keyed on (CPID, Address, IsRemit).
  • Buy/Sell header and lines sync. Daily, including status, ship date, site, AM, line types, posting flags, AR balance, and adjustments.
  • Tax code master sync. Daily, with vendor-CPID mappings (GST, H13, H15, QST).
  • Payment terms master. One-time load with ad-hoc refresh.
  • Vendor resolution baseline. Fuzzy name matching (Levenshtein + token-set ratio), Remit-To exact address match, Ship-From address match, composite confidence with auto-assign threshold.
  • Audit trail foundation. Append-only, hash-chained event log. Seven-year retention. Queryable by document, user, and date range.
  • Per-feed health monitoring. Last-success, next-run, and error-count dashboard. Missing-file detection. One-click re-trigger.
★
Key Benefit

Establishes the data and identity backbone the rest of the platform depends on. After this milestone, vendor matching and Buy/Sell lookup work against live CieTrade data, users can authenticate, and every action is auditable.

Milestone 02

Multi-Channel Invoice Ingestion

Confirmed in commercial proposal
Objective
Ingest every incoming invoice from every source — email, manual upload, and the three vendor portals — into a unified invoice record, with deduplication and source attribution.
Deliverables
Email Ingestion Pipeline
  • MS Graph subscription to APayable@nsrecycle.com with a 60-second poll cadence and a 3-minute end-to-end target.
  • Unread-preservation invariant. AP humans share the inbox; reads must not flip status.
  • Per-attachment invoice record with UUID and source metadata.
  • Supported types: PDF, JPG, JPEG, BMP, TIF, TIFF, PNG, XLS, DOCX, EML.
  • Body-link detection for vendors who email a portal URL instead of attaching the invoice.
  • Supporting document association. CODs, BOLs, and weight tickets auto-linked to the parent invoice.
  • SHA-256 file-hash deduplication on a 30-day window. Critical for AM-forwarded duplicates.
  • Forwarding-chain detection. Use the original sender, not the forwarder.
Manual Upload UI
  • Drag-and-drop multi-file upload.
  • Batch upload for End-of-Month Create Pays.
  • Upload history view. Per-user, 90 days, with status and retry.
Vendor Portal Scraping
  • Playwright headless browser framework with per-portal modules.
  • Waste Connections. Direct PDF download. Sub-brand routing for sub-brands sharing one login surface.
  • GFL. Direct PDF download. Cross-account batch pull.
  • Waste Management. Open-in-new-tab fallback (no direct download). USA and Canada tenant split.
  • Azure Key Vault credential store with hot rotation; password changes do not require restart.
  • Account-number sweep per login (20 to 40 accounts each) with watermark tracking.
  • Retention-aware pagination. Typical portal history is around two years.
  • Email-first, portal-fallback policy. Deduplicated against the email pipeline via SHA-256.
  • Per-portal feature flags for Terms-of-Service and legal compliance gating.
  • Portal failure recovery with alerts, retries, and a per-portal health dashboard.
★
Key Benefit

Every invoice that arrives at NSR, regardless of channel, flows into one pipeline within minutes. AP staff stop manually downloading from portals and stop sorting email.

Milestone 03

Vendor Resolution Engine

Confirmed in commercial proposal
Objective
Resolve every ingested invoice to its correct counterparty (CPID) in CieTrade — including ambiguous, multi-CPID, and unknown-vendor cases — and auto-populate the master-data fields the rest of the pipeline depends on so AP and AM never re-key the same vendor information twice.
Deliverables
  • Vendor Resolution Service. Composite-confidence cascade against the CPID master: fuzzy name match (Levenshtein + token-set ratio), Remit-To exact-address match (IsRemit=1), Ship-From address match (IsRemit=0), and stream-history tiebreak. Auto-assign at composite confidence ≥ 0.95.
  • Address-aware indexing. Solves the "50 Waste Managements" problem. CPIDs indexed on (CPID, Address, IsRemit) tuples so the same vendor name across many counterparties disambiguates by remit-to and ship-from address.
  • Manual Vendor Selection UI. Triggered when composite < 0.95 or when multiple candidates exceed threshold. Ranked candidate list with full match rationale (which signal hit, at what confidence, which addresses matched). One-click assign or escalate.
  • Vendor Not Found queue. Hard-stop fork when no candidate clears threshold. Three resolution paths: assign existing CPID, create new CPID (vendor-master write), or void as not-an-invoice. State machine never silently posts to a wrong account.
  • Master Data Auto-Population. On successful CPID assignment, payment method, payment terms, remit-to address, Sage vendor number, and tax code are pulled from the CieTrade master and written to the invoice record. These flow downstream to validation and posting without re-keying.
  • Learning loop. Manual selections, create-CPID actions, and void decisions are atomically written back to stream history, the address index, and the matcher training set. Subsequent invoices from the same sender or address auto-resolve in one pass.
  • Per-CPID dedupe key. SHA-derived from (CPID + invoice number) and exposed downstream so validation can detect vendor-side duplicate invoices independent of email-level SHA-256 dedupe.
  • Confidence-banded resolution UI. Color-coded badges per resolution signal (name, remit-to, ship-from, stream history). Reviewers see the same rationale the engine used; overrides require a reason captured in the audit log.
★
Key Benefit

Every invoice arrives at the matching engine already keyed to the correct counterparty, with payment terms, remit address, Sage number, and tax code already populated. The "50 Waste Managements" problem stops blocking AP. Unknown vendors hard-stop into a typed queue with a learning loop, instead of silently being misposted.

Milestone 04

OCR and Field Extraction

Confirmed in commercial proposal
Objective
Convert every ingested invoice (PDF, image, Excel, Word, EML) into structured data, with 27 header fields and 5 line fields, per-field confidence scoring, and an editable extraction UI.
Deliverables
  • Pluggable OCR engine. Abstraction layer supporting Azure Document Intelligence, OpenAI Vision, and Tesseract. Per-page OCR with bounding-box retention.
  • 27-field header extractor. Vendor, CPID, invoice metadata, addresses, taxes, payment terms, ship dates, B/S references, and full line-item context.
  • Alias resolution dictionary. Ship Date, B/S Number, Site Location, and Release Number variants normalized to canonical names.
  • Invoice number normalization. Strip leading zeros and apostrophes. Maximum 20 characters.
  • Line-item grid extraction. Sign-preserving for net invoices.
  • Header-vs-line propagation. Ship Date and B/S Number propagate to all lines if captured at the header level.
  • Unit-of-measure normalization. lbs to short tons. kg to lbs. Both stored.
  • Document classification. Invoice vs. Credit Memo (literal string or negative gross). Statement (subject keyword or upload flag).
  • Per-field confidence scoring. Default thresholds: vendor 0.95+, amounts 0.90+, invoice number 0.95+.
  • Low-confidence hold queue. Mandatory-field misses route to Missing Mandatory Info. Nothing silently dropped.
  • Three-pane extraction UI. PDF render, editable fields, B/S context. Click-to-highlight bounding box. Color-coded confidence per field.
★
Key Benefit

Every invoice becomes structured, searchable, and auditable data within seconds of arrival. Confidence is transparent to operators. AP staff stop transcribing invoice fields by hand.

Milestone 05

Buy/Sell Matching Engine

Confirmed in commercial proposal
Objective
Implement the central matching capability of the platform. Link every invoice line to the correct Buy/Sell line at all three observed complexity levels (1:1, N:N, and the harder N:1 grouped case), with full transparency of the match rationale to AP and AM.
Deliverables
  • Header match, 8-step cascade. Direct B/S number match first, then ordered fallback through Release/Vendor, Vendor+Site+Date+Total, and progressively looser combinations.
  • Per-step match rationale. Every match surfaces which step it hit, why, and at what confidence.
  • Confidence scoring per step. Step 1 references at 99 percent, step 8 at 45 percent. Thresholds tunable.
  • Line match, 5-step cascade. Total amount, unit price, qty + UoM, expense line, combined surcharge.
  • Ten line-matching rules. Tax handling, zero-line handling, posted-line exclusion, multi-B/S support, adjustment handling, configurable tolerances.
  • Invoice-total invariant. Sum of matched lines must equal invoice total within tolerance. Failures route to Invoice Not in Balance.
  • Contra and Flip-to-AR logic. Detection, balance check, per-line override with audit, negative chargeback exclusion (Reconserve rule), zero-tolerance enforcement.
  • No-match routing. No-history routes to AP only. Recurring patterns parked 15 days then assigned to AM. Non-recurring assigned immediately.
  • Auto-retry on B/S state change. Transition to Invoiced triggers automatic re-match of all parked invoices for that B/S.
  • Manual match UI. Per-line candidate picker with rationale. Side-by-side invoice + B/S detail. Override with reason and audit.
★
Key Benefit

The single most error-prone step in NSR's current AP process — manually matching invoice lines to spreadsheet-tracked Buy/Sells — becomes deterministic, transparent, and auditable. AM and AP staff see why a match was proposed and can accept, override, or escalate with full context.

Milestone 06

Validation, Workflow, and Exception Management

Confirmed in commercial proposal
Objective
Run every invoice through a deterministic validation gate, route to the right place when checks fail, and give AP, AM, and Manager users structured queues to resolve exceptions efficiently.
Deliverables
  • Eleven pre-posting checks (parallel). Vendor valid, no duplicate, mandatory fields, B/S valid, lines matched, no price/weight variance, B/S = Invoiced, ship-date tolerance, Bill-To match, tax-code match.
  • Hard-stop vs. warning classification. Per-check severity. Warnings continue with surfaced flag; hard stops route to exception queue.
  • Workflow state machine, eleven states. Received through Posted/Paid/Voided. Every transition is an audit event. Multiple exceptions surfaced at once.
  • Ready-for-Submission stabilization gate. auto_post_enabled = false for first three months. Gate-exit criteria: days, auto-post rate, vendor-match rate, regressions, Over-10-Day trend.
  • Twenty exception queue types. AP hard stops, AP warnings, AM routing, monitor queues with auto-retry, Client Services for Completed-status B/S.
  • Queue metadata and operations. Filter, sort, bulk actions. Assignment rules engine. Reassignment UI for AP-to-AM cross-role flows. SLA timers with overdue badging.
  • Auto re-validation on resolution. Upstream changes (B/S transition, vendor refresh, manual edit) trigger dependent validations to re-run automatically.
★
Key Benefit

Every invoice has a clear state at every moment. Every exception has an owner with an SLA. AP and AM Managers can see at a glance where work is piled up. The current Excel-and-email exception triage process is replaced by structured, measurable workflow.

Milestone 07

Special Workflows and Posting Integration

Confirmed in commercial proposal
Objective
Cover the four invoice flows that don't fit the standard Buy/Sell pipeline (Credit Memo, Confidential, Statement Reconciliation, End-of-Month Create Pays), and complete the bi-directional CieTrade SFTP posting integration including payment status reconciliation.
Deliverables
Special Workflows
  • Credit Memo. Dedicated queue, bypasses B/S matching. Manual terminal states. Contra-offset linkage display.
  • Confidential Invoice. Separate mailbox. Sensitivity classification at ingestion. Group-based access enforced server-side. Direct-to-Sage posting path. Daily double-post reconciliation.
  • Statement Reconciliation. Six workflow states. Match key Vendor + Invoice # + Invoice Date. Discrepancy detection. Row-by-row UI. Used for reconciliation, not posting.
  • End-of-Month Create Pays. CP-terms identification. Batch upload on business days 10 and 11. Net positive becomes AP, net negative flips to AR. Atomic batch post.
Outbound Posting (CieTrade SFTP)
  • Posting file generator. Posting_NSR_MMDD.txt, pipe-delimited. Initial fields: UUID, AP_Batch_ID, Invoice_Number, CPID. Full schema locked during integration design.
  • Cadence scheduler. Twice daily, configurable.
  • UUID and source-system tagging. CIETRADE_PATH for standard AP. CONFIDENTIAL_SAGE_DIRECT for confidential.
  • SFTP write with verification. Post-write read-back. Retry on failure. Alert escalation.
  • Outbound audit log. Timestamp, row count, UUIDs, SFTP ACK captured per file.
Inbound Response (CieTrade SFTP)
  • PaymentResponse ingestion. Daily ~09:00. POSTED updates state. FAILED routes to reconciliation queue.
  • PaymentUpdate ingestion. Daily. Reference patterns parsed: -BA ACH, -CK check, -EF EFT. State advances to Paid.
  • "Have You Paid This?" lookup UI. Full timeline per invoice from Received through Paid.
  • Unmatched response handling. Unknown UUIDs route to reconciliation queue. Alert when unmatched volume exceeds 1 percent of daily file count.
  • Per-feed health dashboard. Last success, next run, error count. Failed-file one-click re-trigger. Alerting via email and Slack.
★
Key Benefit

The four hardest invoice flows (credit memos, confidential invoices, statements, create pays) are systematized rather than handled in side processes. The SFTP contract with CieTrade is closed in both directions, so AP staff stop logging into HighRadius to chase payment status.

Milestone 08

Collaboration, Reporting, AR Foundation, and Hypercare

Confirmed in commercial proposal · Includes 3-month hypercare
Objective
Complete the cross-cutting capabilities (collaboration, search, reporting, dashboards), establish the AR Phase 2 foundation, and provide enhanced post-go-live support during the stabilization window.
Deliverables
Collaboration Framework
  • @-tag mentions. Typeahead from Azure AD. Email notifications, real-time or daily digest.
  • Comment scopes. Header, line, statement. Deduction-level deferred to AR Phase 2.
  • On-Hold lock. Rejects concurrent edit, match, post, void. Manager and Admin can release.
  • Reassign-with-comment. Mandatory comment. Full audit trail and notification.
  • Comment-content search. Entitlement-aware. Confidential isolated.
Roles and Permissions (6 roles)
  • AP Processor. Full view, edit, post, void, manual creation.
  • Account Manager. Limited edit. View invoice and B/S lines, update B/S #, manual line-match, comments, reassign. No header edit.
  • AP / AM Manager. All AP capabilities plus cross-role reassign, all queues, full reporting, On-Hold release.
  • Confidential Group A. All confidential including High. No standard queue access.
  • Confidential Group B. Confidential excluding High. No standard queue access.
  • All-Access Admin. Full visibility. Finance management and system administrators.
  • Server-side enforcement. Field-level, queue-level, query-layer. Role changes audited.
Search, Queues, Dashboards
  • Global full-text search. Invoice #, vendor, B/S # on any line, site, ref #, comments, attachment OCR text.
  • Structured filters. Status, queue, vendor, AM, date, age, has-exceptions, currency.
  • Saved searches and custom queues. Per-user or role-shared.
  • Role-tailored dashboards. AP, AM, Manager. Queue counts and aging cards. SLA-breach indicators.
  • Multi-queue membership. A single invoice visible in all applicable queues simultaneously.
Reporting (5 P1 reports)
  • Over-10-Day Report. Highest priority. All invoices not Posted after 10 calendar days. Exportable.
  • Invoice Status Summary. Count and dollar value by stage. Near real-time.
  • Exception / Error Report. All open exceptions, filterable.
  • Aging Report (AP). 0–30, 31–60, 61–90, 90+ buckets.
  • Missing Invoice Report. Recurring B/S patterns versus received. Surfaces stop-shipment risk.
Identity, Security, Compliance
  • TLS 1.2+ on all connections. At-rest encryption on database, blob storage, Key Vault.
  • Confidential segregation. Server-side check on every read. Direct-URL access by unauthorized user returns 403.
  • Penetration testing. Annual cadence. Pre-go-live report delivered to NSR security.
  • SOC 2 Type II attestation, or equivalent, provided pre-cutover.
AR Phase 2 Foundation (sketched)
  • JPMorgan Chase SFTP and myMFT ingestion. Lockbox image ZIPs. EDI 820 X12 v4010 (80-byte wrapped).
  • Cash application match ladder. Invoice #, B/S #, Trace/Reference #, Customer + Amount, Exception.
  • Deductions management with 9-category classification.
  • Unapplied Cash Management. Apply, refund, write-off.
  • Remittance parsing. Email attachments, EDI 820 RMR segments, lockbox image OCR.
  • CAA posting back to CieTrade. Northstar_{customerNo}_{EFT|CHECK}_*.csv per payment event.
Three-Month Hypercare
  • Critical issues acknowledged within one business hour. Resolved or workaround within one business day.
  • Weekly steering meeting with NSR sponsor and TechJays technical lead.
  • Bi-weekly progress report. Touchless rate, exception aging, integration health, outstanding bugs.
  • On-call rotation covering NSR business hours.
  • Post-hypercare retrospective with NSR identifying open scope to roll into Phase 1.5 or Phase 2.
★
Key Benefit

AP, AM, and Manager users have everything they need in one application: communication, search, reports, dashboards. The AR foundation is laid so Phase 2 can begin immediately after Phase 1 sign-off. Hypercare ensures the production rollout is supported through the highest-risk window.
